# 認証機能

## 新規登録時のパスワード要件
- パスワード入力を2回させる
- 入力中リアルタイムで一致判定
- 強度メーター表示
- パスワード表示/非表示トグル
- Firebase側の設定: 小文字と数字を必須

## ユーザー名とユーザーID設定
- 表示名 (displayName): 40文字上限 / 空白不可 / 重複許容
- ユーザーID (handle): 英数字とアンダースコアのみ / 3〜20文字 / 重複不可
- リアルタイム重複チェック（入力停止後 ~450ms）
- 予約語の禁止（admin, support, system 等）
- 大文字を小文字へ変換（ケース非感知）

## プロフィールURL方針
- 正式URL: `/user/{handle}`
- handle で検索し見つからない場合は 404
- 予約語はハンドル登録不可

## 登録フロー
- 未登録状態: / か /login にしかアクセスできない
- 仮登録状態: メール検証リンクをクリック
- 本登録状態: ユーザー名・ユーザーID・パスワードを設定後、全機能にアクセス可能

## パスワードリセット
- Firebase Auth のメールリンク方式
- 中立メッセージで列挙攻撃を防止
- CAPTCHA とレート制限で大量送信を防止
- 詳細は design.md の該当セクション参照
